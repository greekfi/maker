# maker

Python market maker for the Greek.fi options protocol — a port of the node
MM in [`greekfi/protocol`](https://github.com/greekfi/protocol)
(`market-maker/`). Same HTTP/WS surface, same env vars, same Bebop wire
formats — designed as a drop-in replacement. Deribit mode is intentionally
not ported.

## What's different from the node version

- **The pricing seam.** All pricing flows through `PriceSource.price(...)`
  (`greek_mm/pricing/source.py`), an async function that takes every
  parameter needed to price an option (underlying, strike, expiry, is_put,
  chain_id, option_address) and returns a two-sided `PriceResult` in USD
  per 1 unit of underlying notional. The local implementation
  (`SviPriceSource`) prices off an SSVI vol surface + Black-Scholes + fixed
  spreads; swapping in a remote pricing service means implementing the same
  protocol and changing one constructor argument.
- **SSVI surface** (`greek_mm/pricing/svi.py`) replaces the node
  skew/curvature smile. Parameters via `SVI_*` env vars; no-butterfly-
  arbitrage conditions are asserted at startup.
- **Put per-token normalization** (`Pricer.price`): a put option token is
  denominated in consideration collateral — exercising `strike` tokens
  covers one unit of underlying — so quoted prices and greeks are the
  source's notional values divided by strike. The node version skips this
  and quotes puts at full notional price per token.
- **Discovery is unified**: bebop mode uses the same OptionCreated event
  sync + registry as direct mode (the node version's separate metadata.ts
  RPC scan is gone).

## Modes

| Mode | Command | Listens on |
|---|---|---|
| direct | `uv run greek-mm-direct` | `:3010` HTTP, `:3011` WS |
| bebop | `uv run greek-mm-bebop` | none (outbound WS) |
| relay | `uv run greek-mm-relay` | `:3004` HTTP |

## Setup

```bash
uv sync --all-groups
cp .env.example .env && chmod 600 .env
uv run pytest
uv run ruff check .
```

## factories.json

Per-chain factory address + deployment block. The canonical copy lives in
`greekfi/protocol` (`market-maker/factories.json`, generated by
`generateTsAbis.js` on deploy); this repo vendors a snapshot at the root.
Keep it in sync when contracts redeploy, or set `FACTORIES_JSON` to a
protocol checkout's copy.

## Protobuf codegen

Generated bindings are committed (`src/greek_mm/bebop/proto/*_pb2.py`).
To regenerate after editing `proto/*.proto`:

```bash
uv run --no-project --python 3.12 --with grpcio-tools \
  python -m grpc_tools.protoc -Iproto --python_out=src/greek_mm/bebop/proto \
  proto/pricing.proto proto/taker_pricing.proto
```

`proto/taker_pricing.proto` was reconstructed from the node repo's
protobufjs codegen (the original .proto was never committed there).

## Parity notes

- EIP-712 quote signatures are byte-identical to viem's — pinned in
  `tests/test_signing.py::test_signature_matches_node_implementation`.
- Black-Scholes uses exact `math.erf` instead of the node's
  Abramowitz-Stegun approximation (~1e-7 price difference).
- Prices will differ from node by design: the SSVI surface replaces the
  skew/curvature smile, and puts are per-token normalized (see above).
